# Financial Admin Reports & Analysis Proposal

## 1. Executive Summary

This proposal outlines the design and implementation of a financial reporting module for the e-commerce platform's Admin Dashboard. The objective is to provide actionable insights into sales performance through sortable tables and data visualizations, enabling administrators to track revenue, order volume, and product performance over time.

This module leverages existing data models (`Order`, `OrderItem`, `Product`) and extends the Admin API to aggregate financial metrics dynamically.

## 2. Proposed Report Types

The system will feature a dedicated "Reports" tab in the Admin Dashboard, offering three primary views:

### A. Executive Summary
A high-level snapshot of financial performance for a selected date range.

*   **Key Metrics**:
    *   **Orders Count**: Total number of valid orders (excluding `CANCELLED`).
    *   **Gross Sales**: Sum of `subtotal_cents` for all valid orders (revenue before discounts/returns).
    *   **Discounts**: Total value of promo codes and discounts applied (`discount_cents`).
    *   **Returns**: Value of returned items (based on `subtotal_cents` of orders with status `RETURNED`).
    *   **Net Sales**: `Gross Sales` - `Discounts` - `Returns`. Represents pure product revenue.
    *   **Shipping**: Total shipping revenue collected (`shipping_cost_cents`).
    *   **Tax (VAT)**: Total VAT collected (`vat_cents`).
    *   **Total Collected**: Total revenue (`total_cents`), representing Gross Merchandise Value (GMV).

### B. Sales Over Time (Daily Analysis)
A granular view of sales performance broken down by day, allowing trend analysis.

*   **Columns** (Sortable):
    *   **Date**: The date of the transaction (YYYY-MM-DD).
    *   **Orders**: Number of orders placed on that day.
    *   **Gross Sales**: Daily subtotal revenue.
    *   **Net Sales**: Daily net revenue after returns/discounts.
    *   **Shipping**: Daily shipping revenue.
    *   **VAT**: Daily tax collected.
    *   **Total**: Daily total revenue (GMV).

### C. Sales by Item
Performance analysis at the product variant level to identify top-selling items.

*   **Columns** (Sortable):
    *   **SKU**: The unique Stock Keeping Unit identifier.
    *   **Name**: Product name (including variant details like Size/Color).
    *   **Sold**: Quantity of units sold.
    *   **Revenue**: Gross revenue generated by this item (`unit_price * quantity`).

## 3. Technical Implementation

### 3.1. Backend API (`/api/admin/reports`)
The backend will utilize SQLAlchemy aggregation functions to compute metrics efficiently on the database server, minimizing data transfer.

*   **Endpoint**: `GET /api/admin/reports`
*   **Parameters**:
    *   `type`: `summary` | `daily` | `items`
    *   `start_date`: ISO 8601 Date (YYYY-MM-DD)
    *   `end_date`: ISO 8601 Date (YYYY-MM-DD)
    *   `format`: `json` (default) | `csv`
*   **Logic**:
    *   **Filtering**: Filter `Order` records by `created_at` within the date range. Exclude `CANCELLED` orders.
    *   **Aggregation**: Use `func.sum()` and `case()` to aggregate `subtotal_cents`, `discount_cents`, `shipping_cost_cents`, `vat_cents`, and `total_cents`.
    *   **Returns Handling**: Specifically sum values for orders with `status == 'RETURNED'` to calculate the `Returns` metric accurately.

### 3.2. Frontend Interface (`admin_reports.js`)
The frontend will be a Single Page Application (SPA) component within the Admin Dashboard.

*   **UI Components**:
    *   **Date Range Picker**: Allows custom start/end date selection (defaulting to current month).
    *   **Report Type Selector**: Dropdown to switch between report views.
    *   **Data Table**: A dynamic HTML table with sortable headers.
    *   **Export Button**: Triggers a CSV download of the currently viewed data.
*   **Interaction**:
    *   Fetching data via AJAX (`fetch` API) upon clicking "Generate".
    *   Client-side sorting of the table by clicking column headers (handling numeric and date sorting).
    *   Formatting currency values (e.g., converting cents to `€ 10.00`).

## 4. Example Data Flow

**Request:**
```http
GET /api/admin/reports?type=summary&start_date=2023-10-01&end_date=2023-10-31
```

**Response (JSON):**

```json
[
  {
    "orders_count": 150,

    "vat": 85000,            // €850.00
    "total_collected": 550000 // €5,500.00
  }
]
```
